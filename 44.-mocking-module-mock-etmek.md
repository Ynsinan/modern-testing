# 🐿️ 44. Mocking - Module Mock Etmek

Burada ki temel amacımız bir paketi mocklamayı deneyeceğiz.&#x20;

Öncelikle App.js, useFetch.js ve mocks.js adında dosyalarımız olması gerekiyor bunların içerikleri nasıl diye soracak olursak.&#x20;

{% code title="App.js" lineNumbers="true" %}
```javascript
import useFetch from "./useFetch";

function App() {
	const {data,isLoading} =useFetch('https://jsonplaceholder.typicode.com/posts');

	if(isLoading) return <p>Loading...</p>


	return (
		<div>
			{data?.map((post) => (
				<div key={post.id}>
					<h1>{post.title}</h1>
				</div>
			))}
		</div>
	);
}

export default App;
```
{% endcode %}

App.js dosyamızın içinde basic bir request sonucu bir listeleme componentimiz var. Şimdi useFetch.js dosyamıza bakalım.

{% code title="useFetch.js" lineNumbers="true" %}
```javascript
import React, { useEffect, useState } from 'react'

const useFetch = (url) => {
    const [data, setData] = useState(null);
    const [isLoading, setIsLoading] = useState(true);

    useEffect(() => {
        fetch(url).then((response) => response.json())
        .then((data) => {
            setData(data);
            setIsLoading(false);
        })
    }
    , [url]);
  return {data, isLoading}
}

export default useFetch
```
{% endcode %}

Bu şekilde bir url alan yapımız var gelen url sonucu request atıp data ve isLoading parametrelerini dönüyoruz. [Mehmet Pekcan](https://www.linkedin.com/in/mehmetpekcann/) uyarıyor bu tarz bir kullanımı sakın projenize uygulamayın!!! :smile: .

Bundan sonra App.test.js içerisinden devam edeceğiz.&#x20;

Bu noktada ilk önce öğrenmemiz gereken şey modül mocklama işlemi. Bunun için şöyle bir yapı kullanacağız.&#x20;

```javascript
jest.mock("./useFetch", () => {
  return {
    useFetch: () => {
      return {
        isLoading: false,
        data: MOCK_POSTS,
      };
    },
  };
});
```

{% code title="App.test.js" lineNumbers="true" %}
```javascript
import { render, screen } from "@testing-library/react";
import App from "./App";
import { MOCK_POSTS } from "./mocks";

// useFetch modülünü mocklamak için jest.mock kullanılır.
jest.mock("./useFetch", () => {
  return {
    useFetch: () => {
      return {
        isLoading: false,
        data: MOCK_POSTS,
      };
    },
  };
});

describe("<App />", () => {
  it("should be render bla bla", () => {
    // App componentini render ediyoruz.
    render(<App />);

    // Ekranı debug etmek için kullanılan bir fonksiyon. Test sürecindeki elementleri gösterir.
    screen.debug();
    expect(true).toBe(true);
  });
});
```
{% endcode %}

Burada screen.debug çalıştığında terminalimizde mock datamızın ekrana basıldığını görebilirsiniz. Bu noktada bi assertions yapmadık. Şimdi bunun assertion versiyonunu yazalım. Öncelikle App.js dosyamızın içerisinde ul li'li bir yapıya geçelim.&#x20;

Şimdi App.test.js dosyamıza geri dönelim ve assert edelim.&#x20;

{% code title="App.test.js" lineNumbers="true" %}
```javascript
import { render, screen } from "@testing-library/react";
import App from "./App";
import { MOCK_POSTS } from "./mocks";

jest.mock("./useFetch",() => {
  return {
    useFetch: () => {
      return {
        isLoading: false,
        data: MOCK_POSTS,
      }
    }
  };
});

describe("<App />", () => {
  it("should be render bla bla", () => {
    render(<App />);
    
    const listItems = screen.getAllByRole("listitem");
    expect(listItems).toHaveLength(3);
  })
});
```
{% endcode %}

Bu şekilde testimizin success olduğunu ve her hangi bir request atmadan elimizdeki mock data ile bir componenti render edip doğru şekilde çalıştığını doğrulamış olduk.&#x20;
