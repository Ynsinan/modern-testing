# 🦌 47. Dependency Injection - Nedir? Neden İhtiyaç Duyarız?

Dependency Injection (Bağımlılık Enjeksiyonu), yazılım bileşenlerinin diğer bağımlı bileşenleri dışarıdan almasını sağlayan bir tasarım prensibidir. React Testing Library ve Jest bağlamında, bu prensip, bir bileşenin test edilebilir, değiştirilebilir ve sürdürülebilir olmasını sağlar. Bileşenin bağımlılıkları, genellikle props veya context aracılığıyla enjekte edilir, bu da kodun modülerliğini artırır ve gelecekteki değişikliklere daha hazır hale getirir. Jest testlerinde ise, jest.mock() fonksiyonu kullanılarak bağımlılıklar taklit edilir ve testlerde kontrol edilebilir hale getirilir. Bu sayede, kodun daha test edilebilir ve bakımı daha kolay bir hale gelir.

Şimdi kod tarafıyla olayı biraz daha teorik olarak açıklamaya çalışalım. App.js içerisinde liste isteğini ve componentini ayıralım.&#x20;

{% code title="Listing.jsx" lineNumbers="true" %}
```javascript
import { useFetch } from "./useFetch";

function Listing() {
	const {data,isLoading} = useFetch('https://jsonplaceholder.typicode.com/posts');

	if(	isLoading) return <div>Loading...</div>

	return (
		<ul>
			{data?.map((post) => (
				<li key={post.id}>{post.title}</li>
			))}
		</ul>
	);
}

export default Listing;
```
{% endcode %}

Şimdi bu componenti App.js'de çağıralım&#x20;

{% code title="App.js " lineNumbers="true" %}
```javascript
import Listing from "./Listing";

function App() {

	return (
		<div>
			<h1>Title</h1>
			<Listing />
			// Çeşitli componentler...
		</div>
	)
}

export default App;
```
{% endcode %}

Temelde bir componentin içinde bir sürü component kullanabiliriz. Örnek üzerinden bir senaryo canlandıralım. App.js içinde \<Listing /> componentinin App.js üzerinden birden fazla side effecti olabilir istekler, lojikler vs. App componentimizi test ederken Listing componentinin kendi içindeki akışlarını düşünmememiz gerekiyor. Bundan dolayı Listing'in çalışmasını App.js'i test ederken o componenti durdurmak gerekiyor. Sadece componentin geldiğini bilsek yeter. Bu durumda dependencies injectiona başvuruyoruz.&#x20;
