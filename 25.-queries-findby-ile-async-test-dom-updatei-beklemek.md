# 🤽🏽‍♂️ 25. Queries - findBy ile Async Test - DOM Update'i Beklemek

findBy'ın yaptığı en önemli değer önerisi şu getBy ve queryBy'ın aksine asenkron çalışır. Örnek vermek gerekirse dom'da bir noktada Yunus yazıyor. Bir butona tıkladığımda Yunus yazısını Mehmet'e çeviriyorum. getBy ve queryBy onu hala Yunus olarak görüyor bunun sebebi bunlar senkron çalışıyor. Biz asenkron şekilde dom'u takip edecek olursak findBy kullanmamız gerekiyor. Temelde bize bir promise döner ve bulamazsa bize bir hata döner.&#x20;

`findBy`:

* `findBy` fonksiyonları, `getBy` gibi öğeyi belirtilen kritere göre arar, ancak asenkron olarak çalışır.
* Bu fonksiyonlar asenkron işlemleri beklemek için kullanılır. Öğe bulunana kadar bekler ve sonra öğeyi döndürür.
* Eğer öğe belirli bir süre içinde bulunamazsa veya belirli bir zaman aşımı süresi aşılırsa, bir hata fırlatılır ve test başarısız olur.

```javascript
const element = await screen.findByTestId('my-element'); // Asenkron olarak öğeyi bekler.

```

Genel olarak, `getBy` ve `queryBy`, senkron olarak çalışır ve öğe bulunmazsa farklı şekillerde davranırken, `findBy` asenkron olarak çalışır ve belirli bir zaman aşımını bekleyebilir. Bu nedenle, asenkron işlemleri test etmek ve öğeleri beklemek için `findBy` kullanılır



{% code title="App.test.js" lineNumbers="true" %}
```javascript
import { render, screen } from "@testing-library/react";
import React, { useEffect, useState } from "react";

const TestComponent = () => {
  console.log("x");
  const [message, setMessage] = useState("text yok");

  useEffect(() => {
    setTimeout(() => {
      setMessage("text var");
    }, 1000);
  }, []);

  return (
    <div>
      <p>{message}</p>
    </div>
  )
};

it('should render the element correctly"', async () => {
  render(<TestComponent />);
  const element = await screen.findByText("text var");
  expect(element).toBeInTheDocument();

});
```
{% endcode %}

Yukarıdaki örnekte timeout olarak 1 saniye verdiğimizde ve testimizi çalıştırdığımız da terminal ekranında hata alacağız. Bunun sebebi findBy'ın default olarak 1 saniye timeout ile çalışmasından kaynaklanıyor.  Nasıl çözeriz konusunu ileride değerlendireceğiz. O yüzden şimdilik setTimeout'u 1 saniyenin altında tutalım 🥸
