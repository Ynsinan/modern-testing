# 🥁 26. Queries - waitFor ile Async Test Yazmak

findBy ile aynı mental modele sahip bu fonksiyonda async bi callback yazmamıza olanak sağlıyor.

{% code title="App.test.js" lineNumbers="true" %}
```javascript
import { render, screen } from "@testing-library/react";
import React, { useEffect, useState } from "react";

const TestComponent = () => {
  const [message, setMessage] = useState("text yok");

  useEffect(() => {
    setTimeout(() => {
      setMessage("text var");
    }, 300);
  }, []);

  return (
    <div>
      <p>{message}</p>
    </div>
  )
};

it('should render the element correctly"', async () => {
  render(<TestComponent />);
  const element = await screen.findByText("text var");
  const elementShouldNotBe = screen.queryByText("text yok");

  expect(element).toBeInTheDocument();
  expect(elementShouldNotBe).not.toBeInTheDocument();
});
```
{% endcode %}

Temelde şu şekilde bir kodumuz olsun. Günün sonunda bu test başarılı dönüyor. Neden başarılı döndüğünü detaylıca burada açıklamayacağım. Dinlemek isteyen olursa [Mehmet Pekcan Youtube](https://www.youtube.com/watch?v=ciIT-1TcM-E\&list=PLf3cxVeAm43\_2CINQqyUVQCJ94ycC07uz\&index=27) kanalından izleyebilir.&#x20;

Şimdi waitFor kullanmaya çalışırsak.&#x20;

`waitFor`, Testing Library kütüphanesinin bir parçasıdır ve genellikle asenkron işlemleri beklemek ve belirli bir koşulu kontrol etmek için kullanılır. waitFor fonksiyonu, bir koşul gerçekleşene kadar veya belirli bir zaman aşımı süresi geçene kadar beklemeyi sağlar.

```javascript
await waitFor(() => {
  // Beklenen koşulu burada kontrol edin
  // Eğer koşul gerçekleşirse, bu fonksiyon tamamlanır
  // Eğer koşul gerçekleşmezse, bu fonksiyon devam eder ve tekrar kontrol eder
}, options);
```

* İlk parametre olarak bir işlev alır. Bu işlev, belirli bir koşulu kontrol eder. Eğer koşul gerçekleşirse, `waitFor` işlemi tamamlanır. Eğer koşul gerçekleşmezse, fonksiyon tekrar çalıştırılır.
* `options` (isteğe bağlı) parametresi, `timeout` (zaman aşımı) gibi seçenekleri belirlemenizi sağlar.

İlk yazdığımız örneği waitFor kullanarak yazmak istersek&#x20;

{% code lineNumbers="true" %}
```javascript
import { render, screen, waitFor } from "@testing-library/react";
import React, { useEffect, useState } from "react";

const TestComponent = () => {
  const [message, setMessage] = useState("text yok");

  useEffect(() => {
    setTimeout(() => {
      setMessage("text var");
    }, 300);
  }, []);

  return (
    <div>
      <p>{message}</p>
    </div>
  )
};

it('should render the element correctly"', async () => {
  render(<TestComponent />);

  await waitFor(() => {
    expect(screen.getByText('text var')).toBeInTheDocument();
    expect(screen.queryByText('text yok')).not.toBeInTheDocument();
  });
});
```
{% endcode %}

`waitFor` içinde birden fazla kontrol yapmak, karmaşıklığa ve hata olasılığına yol açabilir. Bu nedenle, her bir waitFor içinde bir kontrol yapmak, kodun daha anlaşılır olmasını sağlar ve hata ayıklamayı kolaylaştırır. O yüzden şu şekilde ayırmak daha güzel olacak

```javascript
  await waitFor(() => {
    expect(screen.getByText('text var')).toBeInTheDocument();
    
  });

  await waitFor(() => {
    expect(screen.queryByText('text yok')).not.toBeInTheDocument();
  });
```
