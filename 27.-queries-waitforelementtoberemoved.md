# 🎹 27. Queries - waitForElementToBeRemoved

`waitForElementToBeRemoved`, Testing Library kütüphanesinde kullanılan ve bir öğenin ekrandan kaldırılmasını beklemek için kullanılan bir fonksiyondur. Bu fonksiyon, belirli bir öğenin belirli bir zaman aşımı süresi içinde sayfadan kaldırılmasını bekler ve bu süre sonunda öğe hala ekrandaysa bir hata fırlatır.

`waitForElementToBeRemoved` fonksiyonu, özellikle asenkron işlemleri test ederken veya öğelerin gecikmeli olarak kaldırılmasını beklerken kullanışlıdır. İşte bu fonksiyonun nasıl kullanılacağına dair bir örnek:

{% code title="App.test.js" lineNumbers="true" %}
```javascript
import { render, screen, waitForElementToBeRemoved } from '@testing-library/react';
import React, { useState, useEffect } from 'react';

function MyComponent() {
  const [isLoading, setIsLoading] = useState(true);

  useEffect(() => {
    setTimeout(() => {
      setIsLoading(false);
    }, 2000);
  }, []);

  return (
    <div>
      {isLoading ? <p>Loading...</p> : <p>Data Loaded</p>}
    </div>
  );
}

it('should wait for loading message to be removed', async () => {
  render(<MyComponent />);
  
  // "Loading..." metnini bekliyoruz
await waitForElementToBeRemoved(() => screen.queryByText('Loading...'), { timeout: 10000 });

  // "Data Loaded" metni sayfada olmalı
  expect(screen.getByText('Data Loaded')).toBeInTheDocument();
});
```
{% endcode %}

Bu örnekte, `waitForElementToBeRemoved` fonksiyonu, "Loading..." metninin sayfadan kaldırılmasını bekliyor. İlk olarak, `screen.queryByText('Loading...')` ile bu metni arar ve bu metni beklerken geçici olarak sayfada kalmasına izin verir. Ancak, belirli bir zaman aşımı süresi içinde bu metni bulamazsa, bir hata fırlatır. Duruma göre burada `waitForElementToBeRemoved` için bir timeout verebiliriz.
