# 🎮 32. Kullanıcı Etkileşimleri - Keyboard Events

Temel mantığı pointers ile aynı diyebiliriz. Gerçekten klavyede tuşlara basıyormuşcasına akışımızı ilerletiyoruz. Detaylı [Keyboard API'](https://testing-library.com/docs/user-event/keyboard/)nı buradan inceleyebilirsiniz.

Mevcut counter componentimiz ile devam edeceğiz. Bu aşamada butonumuza tab ile focus olabiliyor muyuz ve daha sonrasında enter tuşuna basarak count'u bir arttırmak istiyoruz.&#x20;

{% code title="App.test.js" lineNumbers="true" %}
```javascript
import { render, screen } from '@testing-library/react';
import userEvent from '@testing-library/user-event';
import { useState } from 'react';

function MyComponent() {
  const [count,setCount] = useState(0);

  const handleIncrement = () => {
    setCount(count + 1);
  };

  return (
    <div>
      <h1>{count}</h1>
      <button onClick={handleIncrement}>Increment</button>
    </div>
  );
}

it('should be increase', async () => {
  // Kullanıcı etkileşimlerini simüle etmek için userEvent.setup() kullanılır.
  const user = userEvent.setup();
  
  // MyComponent bileşeni ekrana render edilir.
  render(<MyComponent />);

  // "Increment" düğmesi, ekranda bulunur ve bu öğeyi alır.
  const buttonElement = screen.getByRole('button', { name: "Increment" });
  
  // Klavye etkileşimi: Klavye olaylarını simüle ederiz.

  // [Tab] tuşu ile düğmeye odaklanmayı simüle ederiz.
  await user.keyboard("[Tab]");
  
  // Düğmenin odaklanıp odaklanmadığını kontrol ederiz.
  expect(buttonElement).toHaveFocus();

  // [Enter] tuşu ile düğmeye basmayı simüle ederiz.
  await user.keyboard("[Enter]");

  // Yeni başlık öğesini bulmak için metin içeriğini bekleriz.
  const headingElement = await screen.findByText('1');

  // Beklenen sonucu test ederiz.
  expect(headingElement).toBeInTheDocument();
});

```
