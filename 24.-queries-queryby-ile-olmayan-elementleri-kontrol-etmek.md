# 🍾 24. Queries - queryBy ile Olmayan Elementleri Kontrol Etmek

Temel farklılık getBy bize elemente ulaşamadığı zaman bize javascript erroru fırlatıyor. Bu bizim istediğimiz bir şey değil. Bundan dolayı ulaşamadığımız bir elementi kontrol etmek istediğimiz zaman `queryBy` kullanmamız gerekiyor.

Örnek vermek gerekirse:&#x20;

{% code title="App.test.js" lineNumbers="true" %}
```javascript
import { render, screen } from "@testing-library/react";

const TestComponent = ({text}) => (
  <div>
    {text && <p>{text}</p>}
    {!text && <p>text yok</p>}
  </div>
);

it('should render the element correctly"', () => {
  render(<TestComponent text="test" />);
  const element = screen.getByText(/test/i);
  expect(element).toBeInTheDocument();
});

```
{% endcode %}

Burada şuan text'i props olarak geçtiğim için bu test success olacaktır.Şuan text'in olduğu durumdan emin olacak şekilde testimizi yazdık. Peki text'in olmadığı durumu nasıl test edeceğiz ?&#x20;

```javascript
  const emptyElement = screen.getByText(/text yok/i);
  expect(emptyElement).not.toBeInTheDocument();
```

Bu satırları ekleyip kontrol edebileceğimizi düşünebiliriz ama yanılıyoruz.

&#x20;Peki doğrusu ne olmalı diye düşünecek olursak. &#x20;

<figure><img src=".gitbook/assets/image (5).png" alt=""><figcaption></figcaption></figure>

```javascript
  const emptyElement = screen.queryByText(/text yok/i);
```

queryBy'ın temelde yaptığı olay şu elemente DOM içinde ulaşamazsa hata fırlatmak yerine null olarak dönüyor. Doğal olarakta bu element var mı yok mu şeklinde kontrol sağlayabiliyoruz. Save ettiğimizde testin başarılı olduğunu göreceğiz.
