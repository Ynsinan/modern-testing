# 🐩 48. Dependency Injection - Implementasyon

Burada yapacağımız teknik implementasyon basit günün sonunda burada nedenini önemini anlamaya çalışacağız.&#x20;

{% code title="App.js" lineNumbers="true" %}
```javascript
import Listing from "./Listing";

function App() {

	return (
		<div>
			<h1>Title</h1>
			<Listing />
			<p>Lorem ipsum</p>
			{/* <Component /> */}
			{/* <Component /> */}
			{/* <Component /> */}
		</div>
	)
}

export default App;
```
{% endcode %}

Burada çeşitli tagler ve componentler varmış gibi düşünün. Bu sıralamanın içinde Listing componentimiz doğru yerde geliyor mu ? İlgilendiğimiz tek soru bu bunun içinde Listing componentinin tamamen render etmek yerine örnek veriyorum boş bir div olarak Listing doğru yere geliyor mu onu test edicez. Bunun içinde listingi bir props haline getirip default halini Listing olarak vermemiz gerekiyor.

{% code title="App.js" lineNumbers="true" %}
```javascript
import Listing from "./Listing";

function App({ListingComponent = Listing}) {

	return (
		<div>
			<h1>Title</h1>
			<ListingComponent />
			<p>Lorem ipsum</p>
			{/* <Component /> */}
			{/* <Component /> */}
			{/* <Component /> */}
		</div>
	)
}

export default App;
```
{% endcode %}

{% code title="App.test.js" lineNumbers="true" %}
```javascript
import { render, screen } from "@testing-library/react";
import App from "./App";

describe("<App />", () => {
  it("should render ListingComponent", async () => {
    // ListingComponent adında sahte bir bileşen tanımlanıyor
    const ListingComponent = () => <div data-testid="listing">Listing</div>;
    
    // App bileşeni render edilirken ListingComponent prop'u olarak sahte bileşen geçiliyor
    render(<App ListingComponent={ListingComponent} />);
    
    // Sahte ListingComponent'in ekranda olup olmadığı kontrol ediliyor
    expect(screen.getByTestId("listing")).toBeInTheDocument();
  });
});

```
{% endcode %}
