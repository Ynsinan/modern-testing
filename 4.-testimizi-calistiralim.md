# 🏁 4. Testimizi Çalıştıralım

Projemizi çalıştırmak için package.json içindeki scriptlerimizden faydalanacağız.&#x20;

{% code title="package.json" lineNumbers="true" %}
```json
  "scripts": {
    "start": "react-scripts start",
    "build": "react-scripts build",
    "test": "react-scripts test", // Bunu kullanacağız.
    "eject": "react-scripts eject"
  },
```
{% endcode %}

Terminalimizi açıyoruz. Daha sonrasında yukarıda yorum satırı ile belirttiğim yerdeki scriptimizi çalıştırıyoruz.

```
yarn test
```

Spesifik olarak bir testin çalışmasını istemediğimiz zamanlarda bu komutu kullanırız ve proje içindeki tüm test dosyalarımız çalışır. Komutu terminale yazıp çalıştırdığımız da aşağıdaki gibi bir görüntü bizi karşılayacaktır.&#x20;

<figure><img src=".gitbook/assets/Screenshot 2023-09-07 at 22.03.12.png" alt=""><figcaption><p>Terminal Ekran Görüntüsü</p></figcaption></figure>

Totalde 1 tane testimiz var ve o da failed oldu. Neden fail oldu sorusunu soracak olursak default içinde yazılı olarak gelen test App.js içindeki kod için yazılmıştır. Fakat biz proje kurulumu ve hazırlanması adımında App.js'imizin içini boşalttığımız için bu noktada testimiz başarısız oldu. Bunun bir önemi yok şuan amacımız test konfigürasyonu ve testlerin çalıştığını doğrulamaktı.&#x20;
