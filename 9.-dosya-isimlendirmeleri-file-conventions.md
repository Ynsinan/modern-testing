# 🗂️ 9. Dosya İsimlendirmeleri - File Conventions

Biz şuana kadar test dosyalarımızı açarken hep `componentName.test.js` şekilinde testlerimizi tanımladık. Jest'e gidip bu bizim test dosyamız demedik. Kendinisi otomatik olarak algıladı ve çalıştırdı. Buna alternatif olarak `componentName.spec.js` olarak da tanımlayabiliriz.&#x20;

**componentName.spec.js:** Alternatif olarak, bazı geliştiriciler bileşen test dosyalarını `.spec.js` uzantısı ile adlandırmayı tercih ederler. Örneğin, `Button.js` bileşeni için `Button.spec.js` adını kullanabilirsiniz.

Bunların yanı sıra en çok karşılaştığımız bir isimlendirme yöntemi de `__test__` klasörü açmak nedir bunun anlamı. Bu durumda, her bileşenin test dosyalarını doğrudan bu klasörde oluşturabilirsiniz. Örneğin, `Button.js` bileşeni için `__tests__/Button.test.js` şeklinde bir dosya yolunu kullanabilirsiniz. Fakat bu \_\_test\_\_ klasörü altında javascript dosyalarımıza test eklememize gerek yok. Yani `__test__/Button.js` olarakta bırakabiliriz.&#x20;

***

3 farklı yöntem gördük. Kullanım şekli açısından birden fazla test dosyasına sahip olmadığımız zamanlar component klasörümüz bu şekilde gözükecek.

```
components/
├── Button/
│   ├── Button.js
│   └── Button.test.js
```

Fakat componentimizin içinde birden fazla test dosyamız olursa&#x20;

```
components/
├── Button
│   ├── __test__/
│   │   ├── test1.js
│   │   └── test2.js
│   └── Button.js
```

Şeklinde gözükmesi daha clean bir code base görmemize sebep olacaktır.&#x20;
