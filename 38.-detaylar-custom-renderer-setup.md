# 🐬 38. Detaylar - Custom Renderer Setup

Çeşitli providerlara sahip olabiliriz. Sayıları çok fazla olur ve en tepe root konumunda olur. Bunun için belirli bir test util dosyaları oluştururuz ve oradan custom setuplarımızı çekeriz. Src altında bir `test-utils.js` dosyası oluşturalım

{% code title="test-utils.js" lineNumbers="true" %}
```javascript
import { render } from "@testing-library/react";

const ThemeProvider = ({ children }) => {
    return <div className="ThemeProvider">{children}</div>;
};

const I18nProvider = ({ children }) => {
    return <div className="I18nProvider">{children}</div>;
};

const AllProviders = ({ children }) => {
    return (
        <ThemeProvider>
            <I18nProvider>{children}</I18nProvider>
        </ThemeProvider>
    );
};
export const customRender = (ui, options) => {
    render(ui, { wrapper: AllProviders, ...options });
};
```
{% endcode %}

Burada çeşitli providerlarımızı yazdık daha sonrasında bir AllProviders oluşturduk. Bunun içerisinde diğer providerlarımızı çağırıp genel bir provider oluşturduk. Daha sonrasında kendi customRender fonksiyonumuzu yazdık. Burada ki options ne diyecek olabilir. Daha sonrası için her hangi bir konuda bir override ihtiyacı duyarsak diye bu kontrolü elimizde tutmuş oluyoruz.&#x20;

Şimdi bunu nasıl kullandığımızı görelim.

{% code title="App.test.js" lineNumbers="true" %}
```javascript
import { screen } from '@testing-library/react';
import { customRender } from './test-utils';

function MyComponent() {
  return (
    <div>
      <p>Modern Testing</p>
    </div>
  );
}

it('should click', async () => {
  customRender(<MyComponent />);

  screen.debug();
});
```
