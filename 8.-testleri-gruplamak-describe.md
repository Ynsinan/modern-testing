# 🧗🏼 8. Testleri Gruplamak - Describe

Bu bölümde test kodlarımızı gruplandırmaktan söz edicez. Yazılım geliştirme sürecinde kodunuzu daha düzenli, okunabilir ve yönetilebilir hale getirmek için kullanılan bir yöntemdir. Kod gruplandırma, aynı amaçları taşıyan veya birlikte çalışan kod parçacıklarını bir araya getirerek organizasyonu artırır ve bakımı kolaylaştırır.

Bu konuyu anlatmak için projemizde bir components klasörü oluşturalım ve altına bir klasör daha açarak "Button" componenti oluşturalım. Son durumda klasör yapım  ve kodlarım şu şekilde.&#x20;

{% code title="/button/index.js" lineNumbers="true" %}
```javascript
const Button = ({children, variant="primary"}) => {
  return (
    <button variant={variant}>{children}</button>
  )
}

export default Button
```
{% endcode %}

Componentimize 2 props geçtik. Children ve variant variant'ın şuan için pek bir önemi yok. Şimdi button klasörümüz içine "button.test.js" adında bir dosya daha açıyoruz. Dosyamızı açtığımız gibi şuan test case'i fail veriyor. Bir test dosyamız var içi boş fakat test yazmadığımız için fail veriyor.&#x20;



Normal şartlar altında bizim variant değerimiz. Primary iken şöyle gözüksün, secondary iken böyle gözüksün gibi gibi senaryolarımız olabilir. Bunu test dosyasına nasıl yazıyoruz?  Bu noktada klasör içerisinde bir gruplandırma gördük. Şimdi ise kod içerisinde nasıl gruplandırabiliriz ona bakalım. Bu konuda bize jestin `describe` metodu yardımcı olacak.&#x20;

```javascript
describe("information text", () => {});
```

{% code title="ornek.test.js" lineNumbers="true" %}
```javascript
describe("Test Grubu Açıklaması", () => {
  // Bu blok içinde yer alan testler Test Grubu Açıklaması altında gruplanır.

  test("Test 1", () => {
    // Belirli bir test senaryosu veya durumu
    // Bu test Grup Açıklaması altında yer alır.
  });

  test("Test 2", () => {
    // Diğer bir test senaryosu veya durumu
    // Yine bu test Grup Açıklaması altında yer alır.
  });
});
```
{% endcode %}

Yukarıdaki örnekte, `describe` fonksiyonu "Test Grubu Açıklaması" adlı bir test grubunu tanımlar. Bu grup altında, "Test 1" ve "Test 2" adlı iki ayrı test senaryosu veya durumu yer alır. Her bir test, `test` fonksiyonu içinde tanımlanır ve bu testler grup içinde gruplandırılmış olur. Aynı zaman da iç içe `describe` kullanımı da mümkündür.

`describe` fonksiyonu, testlerinizi mantıklı bir şekilde düzenlemenize yardımcı olur ve aynı işlevselliği test eden tüm testleri aynı grup altında toplamanızı sağlar. Bu, kodunuzun daha düzenli ve okunabilir olmasına yardımcı olurken, hata ayıklama ve bakım süreçlerini kolaylaştırır.&#x20;



Kodumuzdaki button componentimize göre test bloklarını nasıl şekillendirebiliriz ufak bir örnek olması açısından propslarımıza göre kurgulayabiliriz.&#x20;

{% code title="button/button.test.js" lineNumbers="true" %}
```javascript
describe('Button Component', () => {
   describe('when variant primary', () => {
        test("...", () => {});
        test("....", () => {});
        test(".....", () => {});
    });
    describe('when variant secondary', () => {
        test("...", () => {});
        test("....", () => {});
        test(".....", () => {});
    });
});

```
{% endcode %}

Somut olarak kafamızda bir şey oturmamış olabilir burada odaklanmamız gereken nokta nasıl kurgulayacağımıza dair ufak bir ön gösterim ve `describe` metodunun nasıl kullanıldığı ile alakalıydı. Teorik kısımı geçtikten sonra taşlar yavaş yavaş yerine oturacaktır.
