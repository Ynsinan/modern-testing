# 🏋🏽‍♀️ 5. Testin Anatomisi - Arrange - Act - Assertion

Testin anatomisini incelemeye devam ediyoruz. Testin anatomisinden kastımız şu biz bir Javascript kodu yazıyoruz ama bunu test yapan şeyler ne ? Test dediğimiz şeyler nelerden oluşuyor bunun temelini göreceğiz.

{% code title="App.test.js" lineNumbers="true" %}
```javascript
test("testin açıklaması", 
  () => {
  // Bu alanda test kodları yazılır.
  }
,1000) // timeout süresi default 5 saniyedir
```
{% endcode %}

test() fonksiyonunu açtıktan sonra fonksiyonumuz 3 parametre alır. Birinci parametremiz çalışacak testin açıklaması amacını yazdığımız bölümdür. İkinci parametrede test kodlarımızı yazarız. Üçüncü parametre ise timeout süremizdir.  Ek olarak test() yerine it() de kullanabiliriz.&#x20;

{% code title="App.test.js" lineNumbers="true" %}
```javascript
it('renders learn react link', () => {
  render(<App />);
  const linkElement = screen.getByText(/learn react/i);
  expect(linkElement).toBeInTheDocument();
});
```
{% endcode %}

***

Şimdi test dosyamızdaki default gelen örneği inceleyelim.

{% code title="App.test.js" lineNumbers="true" %}
```javascript
test('renders learn react link', () => {
  render(<App />); // Bu aşamaya teorik olarak Arrangement aşaması denir.
  const linkElement = screen.getByText(/learn react/i); // Act Aşaması 
  expect(linkElement).toBeInTheDocument(); // Assertion aşaması.
});
```
{% endcode %}

Bu kod içindeki test ve expect otomatik olarak import ediliyor. Bunlar normalde Jest'in yardımcı fonksiyonları. Fakat create-react-app sayesinde bunlar default olarak global olarak import ediliyor. Spesifik olarak import ihtiyacı duymuyoruz.&#x20;

Şimdi bu kod neyi ifade ediyor diyecek olursak. RTL içinden render metodunu çağırıyoruz ve App'i render etmesini istiyoruz. Daha sonrasında ekrandaki "learn react" textini bul. Daha sonra expect diyerek bir assertion işlemi yapıyoruz ardından bu elementin DOM'da olmasını beklediğimizi ifade ediyoruz.&#x20;

> Assertion ne demek diyecek olursak. Yazılım geliştirme sürecinin bir parçası olarak test yazılırken "assertion" (iddia) terimi, yazılımın belirli bir koşulu veya davranışı karşılayıp karşılamadığını kontrol etmek için kullanılan bir ifadedir.

> **Eylem (Action):** "Act", bir şeyi yapma veya gerçekleştirme eylemini ifade eder.
