# 🧑🏽‍🔬 13. Queries - getByRole

Bu noktada kodlarımızı test file içinde yazıcaz. Ekstra importing işlemi ile uğraşmamak adına.&#x20;

**getByRole** isminden de açık olduğu üzere elementi rolüne göre select etmeme imkan veriyor.&#x20;

{% code title="App.test.js" lineNumbers="true" %}
```javascript
import { render, screen } from '@testing-library/react';

const TestComponent = () => <button>Button</button>;

it('should render the element correctly"', () => {
  render(<TestComponent />);
  const element = screen.getByRole("")
} );
```
{% endcode %}

İmlecimizi `getByRole` parantezi içerisinde bıraktığımız da aşağıda gördüğümüz üzere çeşitli rolleri bize otomatik olarak öneriyor.&#x20;

<figure><img src=".gitbook/assets/Screenshot 2023-09-23 at 12.42.02.png" alt=""><figcaption><p>Otomatik Öneri Penceresi</p></figcaption></figure>

Şuan yazdığımız TestComponentimizin içinde bir buton var ve bu buton default olarak button rolune sahip. Genel olarak roller konusunda detaylara hakim olmak isterseniz MDN'in bu konu hakkında güzel bir dökümanı var [buradan](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Roles) ulaşabilirsiniz.

Devam edelim yukarıda verdiğim kod parçasında role olarak butonu get etmek istiyoruz.&#x20;

```javascript
 // Burada screen'e git ve buton rolüne sahip elementi al.
 const element = screen.getByRole("button")
 // ve expect ettik. 
 expect(element).toBeInTheDocument();
```

Eğer testComponentimizi bir buton değil de şu şekilde bir div yapacak olursak;&#x20;

```javascript
const TestComponent = () => <div>Button</div>;
```

Bu senaryoda div bir buton rolüne sahip olmadığı için hata alacağız. Fakat bu şekilde de testimizi pass etmemizin bir yöntemi var fakat bu semantic açıdan pek hoş bir kullanım değil.&#x20;

```javascript
// Bu şekilde bir rol ataması yaparsak testlerimiz fail olmayacaktır.
const TestComponent = () => <div role='button'>Button</div>;
```

Şimdi bir senaryo deneyelim componentimizde buton rolüne sahip 2 tane element olsaydı ne yapardık.

```javascript
const TestComponent = () => (
	<>
		<button>Button 1</button>
		<button>Button 2</button>
	</>
);
```

Componentimizi bu hale getirdiğimiz gibi test case'imiz hata atmaya başlayacak. Neden diye soracak olursak getByRole fonksiyonumuz birden fazla elementi select ederse yani seçerse hata fırlatıyor. Peki nasıl 1 tane elementi seçeceğiz. Bunun yolu da basit&#x20;

```javascript
import { render, screen } from "@testing-library/react";

const TestComponent = () => (
	<>
		<button>Button 1</button>
		<button>Button 2</button>
	</>
);

it('should render the element correctly"', () => {
	render(<TestComponent />);
	// Burada rolü buton olan ve name'i Button 1 olan elementi seçmesini istedik.
	const element = screen.getByRole("button",{name: "Button 1"}); 
	expect(element).toBeInTheDocument();
});

```

Daha farklı neye göre seçebiliriz diye soracak olursanız da [buradan](https://testing-library.com/docs/queries/byrole/) ulaşabilirsiniz.
