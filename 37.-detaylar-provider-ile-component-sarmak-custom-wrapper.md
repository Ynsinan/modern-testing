# 🐫 37. Detaylar - Provider ile Component Sarmak - Custom Wrapper

Custom wrapperler günlük hayatta çok fazla kullandığımız yapılardan biri. Temelde context api kullanıyoruz ve ona bağlı bir providerımız var. Böyle bi durumda buna bağlı bir componenti test etmemiz gerekecek fakat providerı olmadığı için soru yaşayacağız. Custom wrapper yapısı kullanmadan da yapabiliriz fakat önerilen yapı bunu custom olarak yapmak. Örneklendirmeye geçecek olursak&#x20;

Öncelikle, `WrapperComponent` adlı bileşeni inceleyelim. Bu bileşen, içine aldığı children prop'ını bir

içerisine koyarak basit bir sarma işlemi yapar. Bu, `MyComponent'i` sarmak için kullanılacak.

{% code title="WrapperComponent.js" lineNumbers="true" %}
```javascript
import React from 'react';

function WrapperComponent({ children }) {
  return <div>{children}</div>;
}

export default WrapperComponent;

```
{% endcode %}

Şimdi, `MyComponent` adlı basit bir bileşenimiz var. Bu bileşen, içinde sadece bir

elemanı bulunduran basit bir yapıya sahiptir.

{% code title=" MyComponent.js" lineNumbers="true" %}
```javascript
import React from 'react';

function MyComponent() {
  return (
    <div>
      <p>Modern Testing</p>
    </div>
  );
}

export default MyComponent;

```
{% endcode %}

Şimdi, `MyComponent` bileşenini WrapperComponent ile sarmak için, test dosyanızda render fonksiyonunu kullanırken `wrapper` prop'unu kullanabilirsiniz.

{% code title="MyComponent.test.js" lineNumbers="true" %}
```javascript
import React from 'react';
import { render, screen } from '@testing-library/react';
import WrapperComponent from './WrapperComponent';
import MyComponent from './MyComponent';

it('should click', async () => {
  render(<MyComponent />, {
    wrapper: WrapperComponent,
  });

  screen.debug()
});

```
{% endcode %}

Bu şekilde, MyComponent bileşenini test ederken onu WrapperComponent ile sarmış olursunuz. Bu sarma işlemi, gerçek projelerde birden çok bileşeni bir araya getirerek ve onların etkileşimlerini test ederek kullanışlıdır.&#x20;
