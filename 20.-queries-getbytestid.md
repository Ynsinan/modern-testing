# 👨🏽‍🎓 20. Queries - getByTestId

Bu işlev, belirli bir test kimliği (test ID) atanan bir HTML öğesini seçmek için kullanılır. Test kimliği, genellikle belirli bir öğeyi test etmek veya testlerde kullanmak için eklenen özel bir `data-testid` özniteliği (attribute) ile ilişkilendirilir.

`getByTestId` işlevini kullanarak bir örnek gösterelim:

```javascript
import { render, screen } from "@testing-library/react";

const TestComponent = () => (
	<div>
		<button data-testid="submit-button">Submit</button>
		<p data-testid="message">Hello, World!</p>
	</div>
);

it('should find the button by its test ID', () => {
	render(<TestComponent />);
	const button = screen.getByTestId("submit-button");
	expect(button).toBeInTheDocument();
});

it('should find the paragraph by its test ID', () => {
	render(<TestComponent />);
	const paragraph = screen.getByTestId("message");
	expect(paragraph).toBeInTheDocument();
});

```

Yukarıdaki örnekte, `getByTestId` işlevi, `data-testid` özniteliği ile belirlenmiş olan "submit-button" ve "message" test kimliklerine sahip öğeleri seçer. Bu işlev, testlerinizi daha spesifik ve güvenilir hale getirmenize yardımcı olur, çünkü HTML yapısına bağımlı olmadan öğeleri test etmenize olanak sağlar.

`data-testid` özniteliği genellikle karmaşık veya dinamik sayfalarda öğeleri daha kolay tanımak için kullanılır. Ayrıca, stil değişikliklerinin veya bileşen yapısının güncellenmesinin testleri etkilemesini engellemek için de kullanışlıdır.

Özetle, `getByTestId` işlevi, belirli bir `data-testid` özniteliği ile işaretlenmiş olan HTML öğelerini seçmek ve test etmek için kullanılır. Bu, testlerinizi daha güvenilir hale getirmenize ve bileşenlerinizi bağımsız olarak test etmenize yardımcı olabilir.
