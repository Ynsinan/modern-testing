# 🎳 29. Debugging - DOM'u Yazdırmak

Ufak bir debugging yönteminden bahsedelim.&#x20;

Bazen caselerimizde neyi render ettiğimizi merak ettiğimiz anlar olabilir. Bunun için container'ı loglayabiliriz. Fakat react elementini komple verdiği için anlaması biraz zor. Nasıl bir log attığını merak ederseniz.&#x20;

```javascript
const { container } = render(<TestComponent />);
console.log(container);
```

Şeklinde loglarsak terminalde karışık bir log attığını göreceksiniz.&#x20;

Daha anlaşılır bir log için `screen.debug()` kullanacağız.

{% code title="App.test.js" lineNumbers="true" %}
```javascript
import { render, screen, waitForElementToBeRemoved } from '@testing-library/react';
import React, { useState, useEffect } from 'react';

function MyComponent() {

  return (
    <div>
      <p className='primary-text'>Modern Testing</p>
    </div>
  );
}

it('should be debug', async () => {
  render(<MyComponent />);
  screen.debug();
  const element = screen.getByText(/Modern Testing/i);
  expect(element).toBeInTheDocument();

});
```
