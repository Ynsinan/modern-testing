# 👮🏾‍♂️ 12. Query'leri Tanıyalım

Dökümantasyon üzerinden derinlemesine detayları keşfedebilirsiniz. Bu konuda bende dökümantasyon üzerinden ilerleyeceğim. [Buradan](https://testing-library.com/docs/queries/about/) ulaşabilirsiniz.&#x20;

Bu kütüphane, kullanıcıların uygulamanın davranışını simüle etmelerini ve sonuçları sorgulamalarını sağlar. RTL içinde kullanılan önemli bir kavram, sorgu işlemleridir. Sorgular, uygulama içindeki bileşenlere erişmek ve onlarla etkileşimde bulunmak için kullanılır. İşte React Testing Library'deki sorguları anlatan temel kavramlar:

**getBy**: `getBy` fonksiyonları, ekrandaki bir öğeyi bulmak için kullanılır. Eğer belirtilen öğe bulunmazsa veya birden fazla öğe bulunursa bir hata fırlatır. Örnek olarak:

{% code lineNumbers="true" %}
```javascript
import { render } from '@testing-library/react';
import MyComponent from './MyComponent';

test('MyComponent içindeki bir başlık öğesini bul', () => {
  const { getByText } = render(<MyComponent />);
  const titleElement = getByText('Başlık');
  expect(titleElement).toBeInTheDocument();
});
```
{% endcode %}

Yukarıdaki örnekte `getByText` kullanılarak bir başlık öğesi (`<MyComponent>` bileşeninde) bulunmuş ve `toBeInTheDocument` eşleştirici kullanılarak bu öğenin sayfada bulunduğu doğrulanmıştır.

**queryBy**: `queryBy` fonksiyonları, ekrandaki bir öğeyi bulmaya çalışır, ancak öğe bulunamazsa `null` döner. Hata fırlatmaz. Bu, bir öğenin sayfada bulunup bulunmadığını kontrol etmek için kullanışlıdır. Örnek olarak:

{% code lineNumbers="true" %}
```javascript
import { render } from '@testing-library/react';
import MyComponent from './MyComponent';

test('MyComponent içindeki bir öğeyi sorgula', () => {
  const { queryByText } = render(<MyComponent />);
  const nonExistentElement = queryByText('BulunmayanElement');
  expect(nonExistentElement).toBeNull();
});

```
{% endcode %}

Yukarıdaki örnekte `queryByText` kullanılarak bulunmayan bir öğenin sorgulanması sağlanmıştır.

**findAllBy ve findBy**: Asenkron işlemlerde kullanılırlar. `findAllBy` birden fazla öğeyi bulurken, `findBy` yalnızca bir öğeyi bulur. Örnek:

{% code lineNumbers="true" %}
```javascript
import { render, waitFor } from '@testing-library/react';
import MyComponent from './MyComponent';

test('MyComponent içinde asenkron bir öğeyi bul', async () => {
  const { findByText } = render(<MyComponent />);
  const asyncElement = await findByText('AsenkronElement');
  expect(asyncElement).toBeInTheDocument();
});

```
{% endcode %}

Yukarıdaki örnekte `findByText` kullanılarak asenkron olarak bir öğenin bulunması sağlanmıştır.

React Testing Library'deki sorgu fonksiyonları, bileşenlerinizi test etmek ve kullanıcı etkileşimlerini simüle etmek için oldukça güçlüdür. Bu fonksiyonları kullanarak bileşenlerinizin beklenen şekilde davrandığını doğrulayabilir ve testlerinizi oluşturabilirsiniz.
